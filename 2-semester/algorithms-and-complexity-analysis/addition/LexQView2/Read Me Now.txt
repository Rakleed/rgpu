Lexicon Quick View
~~~~~~~~~~~~~~~~~~

Автор: Демидов М.В.
Версия программы: см. в конце документа раздел «История версий»

1. Что это такое?
  Программа для «быстрого» просмотра файлов в формате текстового процессора ЛЕКСИКОН для DOS (версий до 1.4 включительно). Программа отображает текст, отформатированный посредством ЛЕКСИКОНа, в окне Windows, используя стандартные экранные шрифты ЛЕКСИКОНа. Кроме того, программа может сохранять текст в файлы формата Windows (с потерей шрифтов и другого форматирования) и копировать фрагменты файлов в буфер обмена Windows в виде рисунка или текста (тоже с потерей шрифтов и форматирования и в формате Windows). Разумеется, с помощью этой программы можно также просматривать и конвертировать в формат Windows обычные текстовые файлы в формате DOS.

2. Кому это нужно?
  Тем, кто вынужден читать файлы в формате ЛЕКСИКОН для DOS, но при этом не хочет запускать саму эту программу (или вовсе не имеет ЛЕКСИКОНа как такового).

3. Как этим пользоваться?
  Нужно запустить исполнимый файл LexQView.exe и открывать файлы с помощью меню (пункт File¦Open...) или комбинации клавиш Ctrl+O (латинская O — русская Щ). [Заметим, что все сочетания клавиш, доступные при работе, отображаются по нажатию F1.] Другой вариант заключается в том, чтобы «перетаскивать» мышью файлы текстов в окне Проводника Windows на значок LexQView. Третий способ — связать файлы определённого типа с этой программой.
  
  Пример: допустим, что в Вашей системе имена файлов ЛЕКСИКОНа имеют расширение *.lex. Тогда можно сделать следующее: в меню Проводника или любой папки найти пункт Вид¦Свойства... В появившемся диалоговом окне должно быть три страницы: «Общие», «Вид» и «Типы файлов». Выбираем последнюю («Типы файлов»), находим в списке слева название, под которым фигурируют в системе файлы с расширением lex, например, «Текстовый документ ЛЕКСИКОНа». Затем нажимаем экранную кнопку «Изменить...» и в возникшем диалоге находим раздел «Действия». Под списком зарегистрированных действий должна быть кнопка «Создать...», которая отображает ещё один диалог. В этом последнем диалоге должны быть две строки ввода: «Действие» и «Приложение». В первой строке вводим «Б&ыстрый просмотр» (именно так, с амперсандом), а рядом со второй должна быть кнопка «Обзор...», с помощью которой мы можем найти на диске файл LexQView.exe. Последовательно нажимаем «Ok» на всех диалогах по очереди. Если всё сделано правильно, то в контекстном меню файлов данного типа будет фигурировать быстрый просмотр.

4. Настройка
  Для настройки используется файл LexQView.cfg, который находится в одной папке с программой. Файл содержит следующие параметры:

    [Settings]
    # Цвет фона. Допустимы названия цветов или значения в формате
    # BGR, в десятичной форме или 16-ричной (в последнем случае
    # нужно указывать знак $ перед числом). Примеры:
    #   White (белый, $FFFFFF=16777215);
    #   Silver или LtGray (светло-серый, $C0C0C0=12632256);
    #   MedGray (средний серый, $A4A0A0);
    #   Gray или DkGray (серый, $808080);
    #   Black (черный, $000000);
    #   Maroon (темно-красный, $000080);
    #   Red (красный, $0000FF);
    #   Green (темно-зеленый, $008000);
    #   Lime (кислотно-зеленый, $00FF00);
    #   MoneyGreen (светло-зеленый, $C0DCC0);
    #   Olive (темно-желтый, $008080);
    #   Yellow (желтый, $00FFFF);
    #   Navy (синий, $800000);
    #   Blue (ярко-голубой, $FF0000);
    #   SkyBlue (небесно-голубой, $F0CAA6);
    #   Purple (фиолетовый, $800080);
    #   Fuchsia (ярко-сиреневый, $FF00FF);
    #   Teal (темно-бирюзовый, $808000);
    #   Aqua (ярко-бирюзовый, $FFFF00);
    #   Cream (кремовый, $F0FBFF).
    # Допускается также указывать цвет стандартных элементов
    # управления Windows, например:
    #   Background (рабочий стол);
    #   Window (окно, этот цвет фона использует Блокнот);
    #   WindowText (текст окна, это цвет шрифта Блокнота);
    #   AppWorkSpace (рабочая область);
    #   Highlight (фон подсветки текста);
    #   HighlightText (подсвеченный текст);
    #   BtnFace (кнопка);
    #   GrayText (серый текст);
    #   BtnText (подпись на кнопке);
    #   InfoText (текст всплывающей подсказки);
    #   InfoBk (фон всплывающей подсказки);
    #   HotLight (текст гиперссылок)
    Background.Color=Window
    # Цвет шрифта, задается точно так же
    Font.Color=WindowText
    # Цвет фона для номеров строк, задается точно так же
    LineNumbers.Background.Color=Window
    # Цвет шрифта для номеров строк, задается точно так же
    LineNumbers.Font.Color=WindowText

    # Масштаб по умолчанию
    Scale=100

    # Антиалиазинг (уменьшение зубцов при масштабировании).
    # Возможны три значения:
    # None — нет, то есть все зубцы видны
    # Simple — простой антиалиазинг, хорошо работает, если
    #     масштаб 200% и больше
    # Oversample — двухступенчатый антиалиазинг с увеличением
    #     и уменьшением, самый медленный способ, хорошо работает,
    #     когда масштаб лежит между 100% и 200%
    Antialiasing=Simple

    # Показывать ли номера строк сразу при загрузке (0 или 1)
    ShowLineNumbers=0
    # Запускать ли сразу на полный экран (выход из полноэкранного
    # режима клавишей F11)
    FullScreen=0

5. Системные требования
  На диске программа занимает около 297 Кб вместе со всеми вспомогательными файлами, в том числе экранными шрифтами ЛЕКСИКОНа (см. ниже), или порядка 204 Кб без них. У программы имеется лишь одно критическое требование: в одном каталоге с LexQView.exe должны присутствовать файлы экранных шрифтов ЛЕКСИКОНа (с расширением sfn). Если ни одного файла не найдётся, то запустить программу не удастся. Где их найти:
 1) в комплекте поставки программы они, вообще говоря, предполагаются, но если их там нет, то
 2) в собственном каталоге ЛЕКСИКОНа, если он у Вас установлен (их можно позаимствовать и с другого компьютера). В последнем случае (если в комплекте поставки Вы не нашли ничего, кроме exe-файла и этого текста) самым простым решением будет разместить exe-файл прямо в каталоге ЛЕКСИКОНа. Это гарантирует, что программа будет отображать тексты в точности так же, как их отображает сам ЛЕКСИКОН.

6. Предупреждение нареканий со стороны пользователей, возможные проблемы при работе программы
  1) Программа работает в графическом режиме: то, что отображается в окне программы — не более, чем «картинка», а не текст. Начиная с версии 2.0 его можно не только просматривать, но и сохранять в текстовом формате и копировать в буфер обмена. При этом пропадают все шрифтовые выделения и специальные символы. Чтобы конвертировать текст один-к-одному, используйте специальные программы-конверторы, которые можно найти в Internet.
  2) Автору случалось наблюдать на некоторых компьютерах с очень старыми версиями Windows (например, Windows'95) неприятные визуальные эффекты при отрисовке экрана (цветная «каша»). Впрочем, это бывает не очень часто (комментарий для посвящённых: проблема связана с тем, что видеосистема такого компьютера недостаточно полно поддерживает функцию Windows BitBlt). Также (на машинах с достаточно «слабой» видеосистемой) окно может перерисовываться слишком медленно, что особенно заметно при быстром пролистывании вверх/вниз. Пользуйтесь вертикальной линейкой прокрутки и клавишами Page Up и Page Down вместо стрелок для более быстрого пролистывания. Масштаб также играет роль: чем он больше, тем медленнее перерисовывается картинка. Номера строк (если включено их отображение) перерисовываются гораздо быстрее, чем изображение документа, поэтому при быстрой прокрутке возможна кратковременная рассинхронизация номеров и соответствующих им строк.
  3) Открывать (в частности, «перетаскивать» на значок программы) можно не более одного текстового файла за один раз.
  4) Нет и не ожидается впредь нормальной справочной системы и многих кажущихся очевидными настроек, как то: печать, переход в режим редактирования (вызов ЛЕКСИКОНа) и т.д., поскольку автор не имеет лишнего свободного времени на подобного рода деятельность.

    * другие мелочи.

  Версия 2.0.1 (18.04.2010):
    * исправлено отображение разделителей страниц, когда цвета фона и текста отличны от принятых по умолчанию;
    * исправлено получавшееся в редких случаях неправильное выделение мышью, в частности, при выходе курсора мыши за пределы окна;
    + добавлено запоминание и восстановление текущей строки при вызове Quck Help и About;
    * существенно расширена документация (т.е. этот файл);
    * улучшен внешний вид;
    * другие мелочи.

  Версия 2.0 (11.04.2010):
    * программа практически полностью переписана, исправлено и изменено множество различных аспектов ее работы;
    + добавлена поддержка колеса мыши;
    + добавлено масштабирование документа с 2 видами антиалиазинга (см. выше пункт «Настройка»);
    + добавлен файл настроек, дающий возможность сохранять сведения о масштабе, цветах символов и фона и т.п. (там же);
    + добавлена возможность копировать данные из документа в буфер обмена Windows в виде рисунка и в виде текста (с потерей всех шрифтов и специальных символов);
    + добавлена функция Save As Text, которая позволяет сохранять документ ЛЕКСИКОНа в виде текста Windows (также с потерей всех шрифтов и специальных символов);
    + добавлена нумерация строк (по умолчанию отключена);
    + добавлен полноэкранный режим по клавише F11 (полезен для владельцев нетбуков и других аналогичных устройств);
    + добавлена возможность открывать из меню Help файл Read Me Now (этот файл), который частично компенсирует отсутствие справочной системы.
